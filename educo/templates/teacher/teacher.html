Notes: For discussion and exercises on bugs, it might be a good idea to work through an example of fixing bugs in a program. Strategies like making test cases to see what it does wrong, and where to put diagnostic print statements to hone in on the error. There’s no exercise related to None but maybe you can chat about it and brainstorm cases in which you would use it. Would be good to point out that to compare to None, you need to use the `is` operator, not `==`. I hope the first exercise makes sense, my main aim is to get students brainstorming corner cases and understanding how they’re important to create when testing some code. In exercise 3, there are default arguments. We haven’t seen them before in tutorials so you might like to point it out for students.
